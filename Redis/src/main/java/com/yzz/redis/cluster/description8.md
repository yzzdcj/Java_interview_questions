###Redis集群
####描述：
    将数据存储在多个Redis节点上，并通过节点间的互相通信实现数据自动分片、故障自动转移、负载均衡，最终达到提升Redis服务的容量和可用性的目的。
####解决方案：
    1.主从集群：主节点写，从节点读，主从一致，可以提高Redis的并发读能力
        全量一致：repulication不一样
        增量一致：repulication一样，offset不同
    2.哨兵；对主从节点的状态进行监控，当主节点挂掉时，会自动切换新的主节点，保证数据一致性，实现高可用
        集群脑裂问题：主节点，从节点，哨兵节点处于不同的网络当中，当哨兵节点与主节点断联，哨兵节点会确定新的主节点，这时会出现两个主节点
        解决办法：设置最少的从节点个数，设置主从数据复制和同步的延迟时间
    3.分片集群：
        多个主节点（高并发写），每个主节点的数据不同（可以存储更多数据），
        每个主节点有多个从节点（高并发读）
        主节点之间相互监控状态（高可用）
        访问任意节点可被路由到对应节点
        存储和访问：16384个哈希槽分配到不同节点，根据key的有效部分确定哈希槽进行存储
